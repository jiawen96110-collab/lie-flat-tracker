# 躺平组合

一个面向移动端与桌面端的多市场投资数据看板，用简洁、清晰的方式集中展示三市场 ETF 组合、周报估值区间和关注标的。

在线访问：

- [Cloudflare Pages](https://lie-flat-tracker.pages.dev/)
- [GitHub 仓库](https://github.com/jiawen96110-collab/lie-flat-tracker)

## 主要功能

### 三市场组合

- 分别展示美股、港股和 A 股 ETF 组合
- 展示持仓权重、当日涨跌和 YTD 表现
- 每 30 秒自动刷新行情，也可手动刷新

### 周报估值

- 美股与 A/H 股分开查看
- 显示每只股票的实时价格、持仓比例和估值位置
- 清晰标注理想买入区、合理区和卖出区
- 根据当前价格突出显示买入信号、等待区或止盈观察
- 针对手机屏幕采用紧凑摘要和大号关键价格，减少无效滚动

估值区间来自周报资料整理，仅作为信息展示。实时行情用于判断当前价格所处区间，不会自动修改周报估值基准。

### 数据整理

- 集中展示关注标的
- 支持按默认分组、累计涨幅和记录日期排序
- 可查看行业、关注逻辑、简要分析和记录价格变化

### 界面体验

- 顶部 Tab 可在组合、周报估值和数据整理之间快速切换
- 同时适配手机和电脑
- 自动跟随系统深色或浅色模式
- 重点数字使用更清晰的字号与层级

## 文件结构

```text
index.html          页面结构与三个主要板块
style.css           全部视觉样式、深浅色主题和响应式布局
app.js              页面交互、行情请求、计算与渲染逻辑
data.js             三市场组合及关注标的数据
valuation.js        美股和 A/H 股周报估值、持仓比例及参考价格
data.json           自动任务生成的行情快照
.github/workflows/  GitHub 自动更新任务
```

项目按内容、样式和逻辑拆分。修改某一类内容时，通常只需要编辑对应文件。

## 如何更新数据

### 更新周报估值

编辑 `valuation.js` 中对应市场的记录：

```js
{
  ticker: 'MRVL',
  sym: 'usMRVL',
  market: 'US',
  name: '迈威尔科技',
  buy: [128.7, 171.6],
  fair: [257.4, 343.2],
  sell: [360.4, 480.5],
  holding: 8.8,
  targetHolding: 14.1
}
```

- `buy`：理想买入区间
- `fair`：合理价格区间
- `sell`：卖出观察区间
- `holding`：当前持仓比例
- `targetHolding`：周报中的参考持仓比例

如果周报只改变持仓比例，通常只需修改 `holding`。如果新增或移除标的，再增删对应记录即可。

### 更新组合或关注标的

编辑 `data.js`。页面布局和行情逻辑一般不需要同步修改。

## 本地预览

这是一个纯静态网页项目，无需安装前端框架。可以直接打开 `index.html`，也可以在项目目录启动任意静态文件服务器进行预览。

## 部署

代码推送到 GitHub 的 `main` 分支后，由 Cloudflare Pages 自动构建并发布：

```text
本地修改 -> 提交到 GitHub -> Cloudflare Pages 自动部署
```

## 数据与风险说明

网页行情主要来自腾讯财经公开接口。第三方接口可能出现延迟、缺失或访问限制，页面展示不应被视为交易依据。

本项目仅用于公开数据整理和个人学习研究，不构成任何投资建议。市场有风险，投资需谨慎。
